# lipsync-monitor

*Іншими мовами: [English](README.md).*

Детектор **розсинхронізації звуку та відео** в реальному часі на основі аналізу
руху губ. Підтримує **UDP-мультикаст** (IPTV / DVB-IP) та **локальні
відеофайли**, повідомляє зсув A/V у мілісекундах і може показувати вікно
попереднього перегляду з виявленим обличчям, губами та звуком.

Виявлення обличчя та лицьових точок виконується на **відеокарті** (NVIDIA, через
InsightFace + onnxruntime-CUDA), тому програма встигає обробляти Full-HD-мовлення
в реальному часі й виявляє малі та не-фронтальні обличчя, які пропускають легші
детектори на CPU.

---

## Як це працює — огляд

```
Вхід (файл / UDP-мультикаст)
        │
        ▼
  ┌─────────────┐     ┌──────────────────────────┐
  │ Кадр відео  │────▶│  LipDetector (GPU)        │──▶ lip_openness[t] ─┐
  └─────────────┘     │  InsightFace SCRFD + 68pt │                     │
                      └──────────────────────────┘             ┌────────▼────────┐
  ┌─────────────┐     ┌──────────────────────────┐             │  SyncDetector   │
  │ Кадр звуку  │────▶│  AudioAnalyzer (RMS)      │──▶ audio_rms[t] ─▶│  крос-кореляц. │─▶ offset_ms
  └─────────────┘     └──────────────────────────┘             └─────────────────┘
```

Захоплення потоку виконується у **фоновому потоці**, який безперервно вичитує
UDP-сокет; якщо детекція не встигає, найстаріші кадри **відео** відкидаються
(звук зберігається), тож сокет ніколи не переповнюється — саме це усуває
«артефакти» (macroblock glitches) на живому потоці.

## Алгоритм детально

### Крок 1 — Сигнал відкриття губ (на GPU)

**Модель:** [InsightFace](https://github.com/deepinsight/insightface), набір
`buffalo_l` — детектор облич **SCRFD** (`det_10g`) + **1k3d68** (68 тривимірних
лицьових точок). Обидві моделі виконує **onnxruntime** на провайдері CUDA (GPU)
з автоматичним відкатом на CPU.

Для кожного кадру:

1. SCRFD виявляє обличчя (роздільність входу за замовчуванням `--det-size 1024`).
   Береться найупевненіше обличчя.
2. Для нього обчислюються 68 лицьових точок (порядок як у dlib).
3. **Відкриття губ** — це відстань між внутрішніми губами, нормована на відстань
   між очима, тому величина не залежить від масштабу:

```
lip_openness[t] = ‖p62 − p66‖ / ‖p36 − p45‖
```

| Точка | Значення |
|---|---|
| **62** | центр внутрішньої верхньої губи |
| **66** | центр внутрішньої нижньої губи |
| **36** | зовнішній кут правого ока ┐ опора масштабу |
| **45** | зовнішній кут лівого ока  ┘ (міжочна відстань) |

Якщо обличчя не виявлено, значення дорівнює `0.0`; якщо все вікно `0.0` — статус
`no_face`.

*Чому InsightFace, а не MediaPipe?* Збірка MediaPipe для Python на Windows працює
лише на CPU, а її легкий детектор пропускає малі й повернуті обличчя, типові для
мовлення (ведучий займає ~5 % кадру 1080p, дивиться вниз на товар). SCRFD на GPU
значно надійніший і обробляє весь потік у реальному часі.

### Крок 2 — Обвідна енергії звуку

**Бібліотеки:** PyAV + NumPy. Кожен декодований аудіопакет ресемплюється у моно
float32 та зводиться до **RMS**-енергії на пакет:

```
audio_rms[t] = sqrt( mean( samples² ) )
```

Величина зростає під час мовлення й спадає під час тиші — дзеркально до відкриття
губ.

### Крок 3 — Крос-кореляція та оцінка зсуву

**Бібліотека:** `scipy.signal`. Обидва сигнали накопичуються у ковзному вікні (за
замовчуванням 3 с) за ключем PTS. На кожному інтервалі звіту:

1. **Ресемплінг** обох сигналів на рівномірну сітку з частотою кадрів
   (`np.interp`).
2. **Перевірка мовлення** — якщо дисперсія губ нижча за `--min-lip-variance`,
   вікно пропускається (`low_speech`).
3. **Z-нормалізація** кожного сигналу.
4. **Крос-кореляція** (`scipy.signal.correlate`) в межах `±--max-offset-search`.
5. **Лаг у піку** кореляції перетворюється на зсув:

```
offset_ms = lag_frames / fps × 1000
```

Довіра (confidence) — це нормований пік кореляції (0–1).

### Домовленість про знак

| `offset_ms` | Значення |
|---|---|
| **додатний** | **звук випереджає відео** — звук чутно *раніше*, ніж рухаються губи |
| **від'ємний** | **звук відстає від відео** — звук чутно *пізніше* за рух губ |

### Допустимі межі — ITU-R BT.1359-1

| Напрям | Межа |
|---|---|
| Випередження звуку | **+45 мс** |
| Відставання звуку  | **−125 мс** |

Стовпець `ITU:` показує `PASS` / `FAIL` відносно цих меж.

## Вимоги

**Апаратне забезпечення (для прискорення на GPU):** відеокарта NVIDIA
(перевірено на RTX 3090) зі встановленими **CUDA 12.x** та **cuDNN 9.x**, які є у
`PATH`. Без GPU програма також працює на CPU (значно повільніше) через CPU-
провайдер onnxruntime.

**Програмне забезпечення:** Python 3.11 (перевірено на Windows 10/11). Версії
пакетів зафіксовано у `requirements.txt`:

| Компонент | Бібліотека |
|---|---|
| Виявлення облич + точки | `insightface` (SCRFD + 1k3d68) |
| Середовище інференсу | `onnxruntime-gpu==1.22.0` (збірка під CUDA 12) |
| Декодування потоку/файлу | `av` (PyAV / FFmpeg) |
| Крос-кореляція | `scipy` |
| Обробка зображень / перегляд | `opencv-python` |
| Числові операції | `numpy<2` |
| Відтворення звуку (опційно) | `sounddevice` |

> **Відповідність onnxruntime / CUDA.** `onnxruntime-gpu==1.22.0` розрахований на
> **CUDA 12** + **cuDNN 9**. Якщо у вас CUDA 13 — візьміть новіший
> onnxruntime-gpu; щоб працювати лише на CPU — замініть його на звичайний
> `onnxruntime`. На Windows Python 3.8+ не шукає залежні DLL у `PATH`, тому
> `src/gpu_setup.py` реєструє каталоги CUDA/cuDNN і попередньо завантажує
> середовище — інакше воно тихо відкотиться на CPU.

## Встановлення

```powershell
git clone https://github.com/distribtech/lipsync-monitor
cd lipsync-monitor

python -m venv .venv
.\.venv\Scripts\Activate.ps1          # PowerShell
pip install -r requirements.txt
```

Під час **першого запуску** InsightFace автоматично завантажує набір моделей
`buffalo_l` (~280 МБ) у каталог `~/.insightface/models`.

На Windows увімкніть UTF-8-вивід, щоб у консолі правильно показувались `…`, `—`:

```powershell
$env:PYTHONIOENCODING = "utf-8"
```

## Використання

```powershell
# Відеофайл
python lipsync_monitor.py -i news_clip.mp4

# UDP-мультикаст (IPTV)
python lipsync_monitor.py -i "udp://@228.0.3.89:1234"

# Вікно перегляду + звук
python lipsync_monitor.py -i "udp://@228.0.3.89:1234" --show --audio

# Поріг тривоги 80 мс, запис у файл, детальна статистика
python lipsync_monitor.py -i "udp://@228.0.3.89:1234" -t 80 -o sync.log -v
```

У **вікні перегляду** (`--show`): **червона рамка** — виявлене обличчя,
**зелена рамка й точки** — губи, блакитний текст — поточний зсув/статус. Вихід —
клавіша **q** або **ESC**.

## Довідник параметрів

| Прапорець | Типово | Опис |
|---|---|---|
| `-i, --input` | *(обов'язково)* | Шлях до файлу **або** `udp://@GROUP:PORT` |
| `-w, --window SEC` | `3.0` | Довжина ковзного вікна аналізу |
| `-n, --interval SEC` | `1.0` | Як часто виводити результат |
| `-t, --threshold MS` | `80.0` | `CRIT`, коли `|offset| ≥ поріг` |
| `--face-confidence` | `0.3` | Мін. довіра детекції обличчя (0–1) |
| `--det-size PX` | `1024` | Роздільність входу детектора (більше = дрібніші обличчя, повільніше) |
| `--min-lip-variance` | `1e-4` | Мін. дисперсія губ, щоб вважати за мовлення |
| `--max-offset-search MS` | `500.0` | Макс. діапазон пошуку зсуву (±мс) |
| `--buffer-size` | `16777216` | Буфер прийому UDP (байт) |
| `--timeout` | `5000000` | Тайм-аут читання з мережі (мкс) |
| `-s, --show` | вимкн. | Вікно перегляду з накладаннями |
| `--show-width PX` | `1280` | Ширина вікна перегляду |
| `-a, --audio` | вимкн. | Відтворювати звук потоку (моно) |
| `-o, --output FILE` | — | Також писати вивід у лог-файл |
| `-v, --verbose` | вимкн. | Детальна статистика по інтервалах |

## Формат виводу

```
Mode: LIVE multicast  |  fps=25.00  |  audio_rate=48000 Hz
LipDetector: InsightFace buffalo_l on GPU (CUDA), det_size=1024
[2026-06-30 14:32:04]  Offset:   +12.0 ms  (audio leads)  |  OK    |  conf: 0.83  |  ITU: PASS
[2026-06-30 14:32:05]  Offset:   -87.5 ms  (audio lags )  |  CRIT  |  conf: 0.91  |  ITU: PASS
[2026-06-30 14:32:06] No face detected in frame
```

| Статус | Значення |
|---|---|
| `OK`   | `|offset| < поріг` |
| `WARN` | `|offset| ≥ 75 % порогу` |
| `CRIT` | `|offset| ≥ поріг` |

**Довіра** (`conf`) — нормований пік крос-кореляції: ≥ 0.7 надійно, 0.4–0.7
помірно, < 0.4 відкинути. Для впевненого показника потрібне обличчя, що говорить,
на екрані щонайменше ~одне вікно аналізу.

---

## Обмеження

- Потрібне **видиме обличчя, що говорить**. Тиша, лише музика, закадровий голос
  або сильний поворот голови дають `no_face` / `low_speech` / низьку довіру.
- «Уривчастий» контент (обличчя на екрані < ~3 с за раз) рідко дає високу довіру
  — найчистіший результат дає стабільний план того, хто говорить.
- **Дубльований** контент за визначенням показує великий зсув (губи ніколи не
  збігаються зі звуком).
- Роздільна здатність зсуву — один кадр відео (40 мс за 25 к/с).

```

├── lipsync_monitor.py    — точка входу CLI, головний цикл
```
